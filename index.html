How to Use This Dashboard

Morning Check (~9 AM)

  1. Glance at the summary bar — green means all clear
  2. Check Main Branch FT Runs — is main green? Multiple failures = systemic issue
  3. Check Cross-PR Spikes — failures across 3+ branches = flaky test or infra issue
  4. Check Allow List Status — any owned tests quarantined? Prioritize fixing them

After Fixing a Flaky Test

  1. Use "Run by Test String" with the test name to validate 5x
  2. Watch the result appear in LAFing Cow Dispatch Runs

Before End of Day

  1. Dispatch a full "Run LAFing Cow FTs" for a clean baseline tomorrow

When You See a Spike

  1. Expand the failed runs in Cross-PR Spikes
  2. Same test failing across unrelated branches = flaky on main
  3. Failures clustered on one branch = real breakage from that PR
